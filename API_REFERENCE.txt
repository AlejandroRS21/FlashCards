API REFERENCE AND TECHNICAL DOCUMENTATION - FlashCards v1.3
============================================================

CORE CLASSES REFERENCE
======================

1. MainActivity.kt

Class: MainActivity : ComponentActivity

Purpose: Application entry point and navigation orchestrator

Key Methods:
- onCreate(savedInstanceState: Bundle): Initialize app and Compose
- setContent: Set Compose UI content

Key Variables:
- currentScreen: MutableState<Screen> - Current visible screen
- decks: MutableState<List<Deck>> - All available decks
- selectedDeck: MutableState<Deck?> - Currently selected deck

Navigation States:
enum class Screen {
    Dashboard, Study, Import
}

Data Flow:
1. Load all decks from DataManager on startup
2. Display Dashboard
3. Navigate to Import or Study based on user action
4. Pass callbacks to child composables
5. Update state on navigation

2. DataManager.kt

Class: DataManager

Purpose: Manage data persistence with SharedPreferences

Constructor: DataManager(context: Context)

Public Methods:

saveDeck(deck: Deck): Unit
- Saves deck to SharedPreferences
- Updates deck list
- Triggers listeners

saveDeck(deckWithFlashcards: DeckWithFlashcards): Unit
- Saves deck with flashcards
- Updates deck list
- Overwrites existing

getDeck(deckName: String): DeckWithFlashcards?
- Retrieves specific deck with flashcards
- Returns null if not found

getAllDecks(): List<Deck>
- Returns list of all decks
- Loads flashcards for each deck
- Includes progress data

deleteDeck(deckName: String): Unit
- Removes deck from storage
- Updates deck list
- Removes associated flashcards

updateDeck(deck: Deck): Unit
- Updates existing deck
- Calls saveDeck() internally

Private Methods:
- updateDecksList(deck: Deck): Update master deck list

3. CSVParser.kt

Class: CSVParser

Constructor: CSVParser(context: Context)

Public Methods:

parseCSV(uri: Uri): Result<List<Flashcard>>
- Parse CSV file from URI
- Returns Result type (success or failure)
- Validates file format
- Returns: Result<List<Flashcard>>

Success Case:
Result.success(flashcards: List<Flashcard>)

Failure Cases:
Result.failure(exception: Exception)

Supported Exceptions:
- FileNotFoundException: File not found
- IOException: File read error
- IllegalArgumentException: Invalid format

Private Methods:
- parseCSVLine(line: String): Flashcard? - Parse single line
- parseCSVFields(line: String): List<String> - Parse CSV fields

CSV Format:
question,answer
"quoted field with, comma","quoted answer"

3. StudySessionScreen Composable

@Composable
fun StudySessionScreen(
    onClose: () -> Unit,
    deck: Deck? = null,
    question: String = "",
    answer: String = "",
    currentCardIndex: Int = 0,
    totalCards: Int = 1,
    onDeckUpdate: (Deck) -> Unit = {}
)

Parameters:
- onClose: Exit session callback
- deck: Deck object with flashcards
- question: Single card question (fallback)
- answer: Single card answer (fallback)
- currentCardIndex: Current position
- totalCards: Total card count
- onDeckUpdate: Callback for deck updates

State Variables:
- currentIndex: Int - Current card index
- isFlipped: Boolean - Front/back state
- showEditDialog: Boolean - Edit dialog visibility
- editedDeckName: String - New name in edit
- offsetX: Animatable<Float> - Swipe animation
- flipRotation: Float - Flip animation value

Animation Objects:
- offsetX: Swipe horizontal movement
- flipRotation: Card 3D rotation

Methods:
- goToNextWithAnimation(): Navigate to next with animation
- goToPreviousWithAnimation(): Navigate to previous with animation
- animateSwipeNext(): Execute next animation
- animateSwipePrevious(): Execute previous animation

EVENT HANDLING
==============

Gesture Events:

Swipe Gesture:
- Type: detectHorizontalDragGestures
- Minimum threshold: 100dp
- Direction: dragAmount > 0 (right), dragAmount < 0 (left)
- Action: Launch coroutine with animation

Tap Gesture:
- Type: clickable modifier
- Action: Toggle isFlipped state
- Animation: 500ms rotation

Button Clicks:
- Type: onClick lambda
- Action: Execute callback immediately
- No debouncing

ANIMATION SPECIFICATIONS
========================

Flip Animation:

Property: rotationY (Y-axis rotation)
Start value: 0 degrees
End value: 180 degrees
Duration: 500 milliseconds
Curve: Cubic tween
Trigger: Tap on card

graphicsLayer {
    rotationY = flipRotation
    cameraDistance = 12f * density
}

Swipe Animation:

Properties:
- translationX: Horizontal movement (±1200px)
- rotationZ: Rotation angle (±25 degrees)
- scaleX/Y: Scale factor (0.9x)
- alpha: Opacity (0.7)

Duration: 400 milliseconds
Curve: Linear tween
Trigger: Swipe gesture completion

animateSwipeNext() sequence:
1. offsetX.animateTo(-1200f, duration 400ms)
2. rotationZ.animateTo(-25f, duration 400ms)
3. offsetX.snapTo(0f) - Reset
4. rotationZ.snapTo(0f) - Reset

Stack Effect:

Layer 0 (Front card):
- translationY: 0px
- scaleX/Y: 1.0
- alpha: 1.0

Layer 1 (Next card):
- translationY: 12px
- scaleX/Y: 0.95
- alpha: 0.9

Layer 2 (Third card):
- translationY: 24px
- scaleX/Y: 0.90
- alpha: 0.8

Kotlin compilation: 3 seconds
Gradle tasks: 2 seconds

File I/O:

CSV parse (100 cards): <500ms
Deck save: <100ms

SECURITY CONSIDERATIONS
=======================

Input Validation:

CSV File:
- Extension validation (.csv)
- File size limits
- Column count validation
- Empty field handling

Deck Name:
- Length limits (1-50 chars)
- Special character handling
- Uniqueness check

Card Content:
- No validation (user content)
- Text length recommendations
- HTML/Script filtering (future)

Data Storage:

SharedPreferences:
- No encryption (local device)
- Device-level security
- File permissions: 0600 (owner read/write)
- User-controlled backup eligibility

Transmission:
- No network transmission
- No cloud synchronization
- 100% local storage

Permissions:

Required:
- READ_EXTERNAL_STORAGE (file access)

Not Required:
- No network permissions
- No camera permission
- No microphone permission
- No location permission
- No contact permission

ERROR HANDLING
==============

Exception Types:

FileNotFoundException
- Thrown when file not found
- Handler: Show error dialog
- Recovery: Allow retry file selection

IOException
- Thrown on file read error
- Handler: Show technical error
- Recovery: Try different file location

IllegalArgumentException
- Thrown on format validation
- Handler: Show format error
- Recovery: Provide format instructions

NullPointerException
- Thrown on null access
- Handler: Default values fallback
- Recovery: Automatic recovery

Coroutine Exceptions:

CancellationException
- Normal coroutine cancellation
- No user notification
- Automatic cleanup

TimeoutException
- Operation timeout (configurable)
- Handler: Timeout error dialog
- Recovery: Retry operation
