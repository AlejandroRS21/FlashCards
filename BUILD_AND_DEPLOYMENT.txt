BUILD AND DEPLOYMENT GUIDE - FlashCards v1.3
==============================================

ENVIRONMENT SETUP
=================

Prerequisites:

1. Android Studio (Latest version 2024.1 or later)
   - Download from: developer.android.com/studio
   - Installation guide: Follow Android Studio setup wizard

2. Android SDK
   - Minimum SDK: API 33 (Android 13)
   - Target SDK: API 34 (Android 14)
   - Ensure SDK installed via SDK Manager

3. Gradle
   - Version: 9.1 (automatic with Android Studio)
   - Located in: gradle/wrapper/gradle-wrapper.jar

4. Java Development Kit
   - Version: Java 11 or later
   - Set JAVA_HOME environment variable
   - Verify: java -version in terminal

5. Kotlin
   - Version: 2.2.0 (automatic with Gradle)
   - Plugin installed via build.gradle

   Example (Linux):
   sdk.dir=/home/username/Android/Sdk

DEVELOPMENT BUILD
=================

Build Debug APK:

1. Using Android Studio:
   a. File > Open > Navigate to project
   b. Click "Open" to load project
   c. Wait for Gradle sync (first build 5-10 minutes)
   d. Build > Build Bundle(s)/APK(s) > Build APK(s)
   e. Locate APK: app/build/outputs/apk/debug/app-debug.apk

2. Using Command Line:
   a. Open terminal in project root
   b. Run: ./gradlew assembleDebug (Windows: gradlew.bat assembleDebug)
   c. Wait for build completion
   d. APK location: app/build/outputs/apk/debug/app-debug.apk

Clean Build (if issues occur):
   ./gradlew clean assembleDebug

Build Output:
   BUILD SUCCESSFUL in 8s
   APK: app-debug.apk (approximately 20MB)

RUNNING ON EMULATOR
===================

Setup Android Virtual Device (AVD):

1. Open AVD Manager in Android Studio
2. Create new virtual device:
   a. Click "Create Virtual Device"
   b. Select device type: Pixel 6 (recommended)
   c. Select target API: 34 (Android 14)
   d. Name device: FlashCards_Emulator
   e. Finish and start device

2. Run Application:
   a. Connect device (physical or emulator)
   b. Click green Run button in Android Studio
   c. Select target device
   d. Click OK to launch app

   Or from command line:
   ./gradlew installDebug
   adb shell am start -n com.ramsalapps.flashcards/.MainActivity

TESTING ON PHYSICAL DEVICE
==========================

Prerequisites:
1. Android device with API 33 or higher
2. USB cable for connection
3. USB debugging enabled on device

Enable USB Debugging:

1. Settings > About > Build Number (tap 7 times)
2. Settings > System > Developer Options
3. Enable "USB Debugging"
4. Connect device to computer via USB
5. Authorize computer when prompted on device

Install Debug APK:

1. Copy app-debug.apk to device
   Method 1: Direct installation
   a. Email APK to device
   b. Open email on device
   c. Download and tap APK
   d. Tap Install

   Method 2: Via ADB
   a. Connect device via USB
   b. Open terminal in project
   c. Run: adb install app/build/outputs/apk/debug/app-debug.apk
   d. Wait for "Success" message

2. Test Application:
   a. Find FlashCards app icon on device
   b. Tap to open
   c. Test all features
   d. Import test CSV file
   e. Create deck
   f. Study cards

RELEASE BUILD
=============

Generate Release APK:

1. Using Android Studio:
   a. Build > Build Bundle(s)/APK(s) > Build APK(s)
   b. Select "release" from configuration
   c. Provide signing key information
   d. Wait for build completion

2. Using Command Line:
   a. Run: ./gradlew assembleRelease
   b. Provide keystore information when prompted
   c. APK location: app/build/outputs/apk/release/app-release.apk

Signing Key Setup:

1. Create Signing Key (first time only):
   a. Build > Generate Signed Bundle/APK
   b. Select APK
   c. Click Create New > Create Key Store
   d. Provide information:
      - Key store path: (choose location)
      - Key store password: (create secure password)
      - Key alias: flashcards_key
      - Key password: (create secure password)
      - Validity: 30 years minimum
   e. Fill certificate information (name, organization, etc.)
   f. Click OK

2. Sign Release APK:
   a. Build > Generate Signed Bundle/APK
   b. Select APK
   c. Select keystore created above
   d. Enter passwords
   e. Select release build type
   f. Finish to generate signed APK

Build Output:
   Signed APK: app/build/outputs/apk/release/app-release.apk
   Size: Approximately 15MB

TROUBLESHOOTING BUILD ISSUES
============================

Issue: Gradle sync fails

Solutions:
1. File > Invalidate Caches > Invalidate and Restart
2. Delete .gradle folder and sync again
3. Update Android Studio to latest version
4. Check internet connection (downloads dependencies)

Issue: Build fails with compilation errors

Solutions:
1. Clean project: ./gradlew clean
2. Check Java version: java -version (should be 11+)
3. Verify SDK installation
4. Check file encoding (UTF-8)
5. Review error message for specific issue

Issue: APK installation fails

Solutions:
1. Uninstall previous version: adb uninstall com.ramsalapps.flashcards
2. Clear data: Settings > Apps > FlashCards > Clear Data
3. Ensure sufficient storage on device
4. Check Android version (13+)
5. Verify APK not corrupted (check file size ~15-20MB)

Issue: App crashes immediately after launch

Solutions:
1. Check logcat for error message
2. Verify permissions in AndroidManifest.xml
3. Ensure all dependencies resolved
4. Check SharedPreferences initialization
5. Review data loading logic

PERFORMANCE OPTIMIZATION
========================

Build Optimization:

1. Enable ProGuard (Release builds):
   - Reduces APK size by ~25%
   - Obfuscates code
   - Improves performance
   - Configuration: proguard-rules.pro

2. Minimize dependencies:
   - Use only required libraries
   - Remove unused code
   - Optimize imports

3. Optimize resources:
   - Remove unused strings
   - Compress images
   - Reduce asset file sizes

Runtime Optimization:

1. Memory management:
   - Avoid memory leaks
   - Close resources properly
   - Use appropriate data structures

2. UI performance:
   - Lazy load lists
   - Efficient animations
   - Minimize recompositions
