FlashCards Application - v1.3 Final Release
===============================================

PROJECT OVERVIEW
================

FlashCards es una aplicación de estudio digital que permite a los usuarios importar
flashcards desde archivos CSV, crear decks temáticos y estudiar con una interfaz
intuitiva basada en gestos. La aplicación implementa un sistema de aprendizaje
interactivo con animaciones fluidas y persistencia de datos local.

TECHNICAL SPECIFICATIONS
========================

Platform: Android 13+ (API 33+)
Framework: Jetpack Compose
Language: Kotlin 2.2.0
Architecture: MVVM with State Management
Data Persistence: SharedPreferences + JSON (GSON)
Build System: Gradle 9.1

KEY FEATURES IMPLEMENTED
========================

1. CSV Import System
   - File selection via Android file browser
   - CSV parsing with validation (question, answer format)
   - Error handling for invalid files
   - Support for UTF-8 encoded files
   - Preview of imported cards before creation

2. Deck Management
   - Create decks from imported CSV files
   - Edit deck names in real-time
   - Delete decks with confirmation
   - Automatic card counting
   - Persistent storage of all decks

3. Study Session Interface
   - Card stack visualization (3 cards visible)
   - Pastel color scheme (6 distinct colors)
   - Tap to flip card animation
   - Swipe gestures for navigation
   - Previous/Next button controls
   - Progress tracking per session

4. Navigation System
   - Bottom navigation bar
   - Dashboard with recent decks section
   - Import screen with multi-step flow
   - Back gesture support

5. User Interface
   - Material Design 3 compliance
   - System bar respect (notch and navigation)
   - Responsive layout
   - Compact component sizing
   - Text overflow handling

6. Data Management
   - Local database via SharedPreferences
   - JSON serialization for data storage
   - Automatic data persistence
   - Deck recovery on app restart

CORE COMPONENTS
===============

1. Data Layer
   File: DataManager.kt
   - Saves/loads decks with flashcards
   - Manages deck list
   - Handles updates to existing decks
   - Data validation

   File: CSVParser.kt
   - Parses CSV file format
   - Validates column structure
   - Returns Result type for error handling
   - Supports quoted fields

2. Model Layer
   File: Models.kt
   - Deck: name, cardCount, progress, icon, flashcards
   - Flashcard: question, answer, category
   - Session: tracking study sessions
   - DeckWithFlashcards: composite object for persistence

3. Presentation Layer

   MainActivity.kt
   - App entry point
   - Screen navigation management
   - Callback handling for screen transitions

   DashboardScreen.kt
   - Home screen with statistics
   - Recent decks section
   - All decks horizontal scroll
   - Daily goal card
   - Start review button

   ImportScreen.kt
   - Two-step import process
   - SelectFileStep: file selection and naming
   - ReviewCardsStep: preview and creation
   - Progress indication during import

   StudySessionScreen.kt
   - Card display with stack effect
   - Flip animation (500ms duration)
   - Swipe gesture detection
   - Navigation controls
   - Deck name editing dialog

4. UI Theme
   File: theme/Theme.kt and theme/Color.kt
   - Material Design 3 color scheme
   - Pastel color palette
   - Typography definitions
   - Light theme only

APPLICATION FLOW
================

User Journey:
1. User opens app - sees Dashboard
2. Click Library button - navigates to ImportScreen
3. Select CSV file - enters deck name
4. Preview flashcards - reviews data
5. Click Create - saves deck and returns to Dashboard
6. Click deck card - enters StudySessionScreen
7. Study using swipe or buttons - flips cards with tap
8. Close session - returns to Dashboard

File Processing:
1. User selects CSV file from device storage
2. App validates file extension (.csv)
3. CSVParser reads and processes file
4. Each line parsed into Flashcard object
5. User enters deck name
6. Deck created with flashcards
7. Data serialized to JSON and saved

DATA STORAGE
============

SharedPreferences Structure:
- Key: "all_decks" → JSON array of Deck objects
- Key: "deck_[name]" → JSON DeckWithFlashcards object

Example Storage:
all_decks: [
  {"name": "Spanish 101", "cardCount": 45, "progress": 30, "icon": "es"},
  {"name": "Biology 201", "cardCount": 120, "progress": 75, "icon": "bio"}
]

deck_Spanish_101: {
  "deck": {"name": "Spanish 101", ...},
  "flashcards": [
    {"question": "Hola", "answer": "Hello", "category": "Imported"},
    ...
  ]
}

ANIMATION SPECIFICATIONS
========================

Flip Animation:
- Duration: 500ms
- Type: Card 3D rotation on Y-axis
- Trigger: Tap on card

Swipe Animation:
- Duration: 400ms
- Translation: ±1200px horizontal
- Rotation: ±25 degrees
- Scale: Gradually reduces to 90%
- Opacity: Fades to 70%

Stack Effect:
- Card spacing: 12px vertical offset
- Scale reduction: 5% per layer
- Opacity reduction: 10% per layer
- Visible layers: 3 cards maximum

ERROR HANDLING
==============

CSV Import Errors:
- Invalid file format: Show error message, allow retry
- Missing columns: Display column count mismatch
- Encoding issues: Display warning, attempt to process
- Empty file: Show "no valid cards found"

Data Persistence Errors:
- Save failure: Log error, notify user
- Load failure: Provide empty list fallback
- Corrupt data: Skip corrupted entries
