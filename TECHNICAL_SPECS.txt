TECHNICAL SPECIFICATIONS - FlashCards v1.3
===========================================

ARCHITECTURE OVERVIEW
=====================

Layer 1: Presentation Layer (UI)
- MainActivity: Activity entry point and navigation orchestration
- DashboardScreen: Home screen composable with deck listing
- ImportScreen: Multi-step CSV import interface
- StudySessionScreen: Card study interface with gesture support

Layer 2: Domain Layer (Logic)
- CSVParser: File parsing and validation logic
- Navigation state management in MainActivity

Layer 3: Data Layer (Persistence)
- DataManager: SharedPreferences wrapper
- GSON serialization for data storage

DETAILED COMPONENT SPECIFICATIONS
==================================

1. MainActivity.kt

Purpose: Application entry point and navigation controller

Responsibilities:
- Initialize Compose with FlashCardsTheme
- Manage screen state transitions
- Handle activity-level callbacks
- Enable edge-to-edge display

Data Flow:
- Screen enumeration: Dashboard, Study, Import
- currentScreen state drives visible composable
- onDeckClick triggers navigation to Study screen
- onBack returns to Dashboard

Navigation Flow:
  Dashboard -----> Import Screen
     |                 |
     |                 v
     +----------> Study Screen

2. CSVParser.kt

Purpose: Parse and validate CSV files

Method: parseCSV(uri: Uri): Result<List<Flashcard>>

Parsing Logic:
1. Open InputStream from URI
2. Read file line by line
3. Parse each line into fields
4. Handle quoted fields with commas
5. Validate field count (minimum 2)
6. Create Flashcard objects

Error Handling:
- Empty file: Returns failure
- Missing columns: Silently skips invalid rows
- Encoding issues: Attempts UTF-8 decoding
- Exception wrapping: All errors in Result.failure()

CSV Format Requirements:
- Field delimiter: comma (,)
- Optional quotes: double quotes ("")
- Expected columns: question, answer (minimum 2)
- Line endings: CRLF or LF

3. DataManager.kt

Purpose: Manage local data persistence

Persistence Strategy:
- SharedPreferences for key-value storage
- JSON serialization via GSON
- One entry per deck + one master list

Key-Value Structure:
{
  "all_decks": "[{Deck}, {Deck}, ...]",
  "deck_[name]": "{DeckWithFlashcards}"
}

Methods:
- saveDeck(deck: Deck): Persist deck and update list
- getDeck(name: String): Retrieve specific deck
- getAllDecks(): Get list of all decks
- deleteDeck(name: String): Remove deck
- updateDeck(deck: Deck): Modify existing deck

Thread Safety: All operations on main thread via Compose

4. DashboardScreen.kt

Purpose: Display home screen with deck overview

Components:
- Header: User greeting with profile icon
- DailyGoalCard: Progress tracker
- StartReviewButton: CTA for studies
- RecentDecksSection: Recently accessed decks
- AllDecksSection: Horizontal scrolling deck list
- BottomNavigationBar: Navigation to Import

Data Binding:
- decks: List<Deck> from MainActivity
- onDeckClick: Callback for deck selection
- onImportClick: Callback for import navigation

UI Constants:
- Card width: 160dp
- Spacing: 16dp between items
- Corner radius: 16dp for cards
- Max lines: 2 for deck names
- Icon size: 40sp

5. ImportScreen.kt

Purpose: CSV import with two-step workflow

Step 1: SelectFileStep
- File browser launch
- Deck name input
- File validation
- Error display
- Recent files list

Step 2: ReviewCardsStep
- Flashcard preview table
- Card count display
- Deck name confirmation
- Create button

State Management:
- importStep: Current step (SELECT_FILE or REVIEW_CARDS)
- selectedFileUri: Chosen file reference
- parsedFlashcards: List of parsed cards
- errorMessage: User-facing error text
- isLoading: Loading state during parsing

6. StudySessionScreen.kt

Purpose: Interactive card study session

Card Stack Display:
- Layer 0 (Front): Current card, full opacity
- Layer 1 (Back): Next card, 90% opacity, 95% scale
- Layer 2 (Back): Card after next, 80% opacity, 90% scale
- Beyond: Hidden

Gesture Handling:
- Swipe detection: detectHorizontalDragGestures
- Minimum threshold: 100dp movement
- Direction: Positive = right, Negative = left
- Animation: animateSwipe() function with tween

Animation Details:
Flip Animation:
- Property: rotationY (0-180 degrees)
- Duration: 500ms
- Curve: Cubic tween
- Trigger: Card tap

Swipe Animation:
- Properties: translationX, rotationZ, scaleX/Y, alpha
- Duration: 400ms
- Start: User drag release
- Direction-dependent movement

Color Assignment:
- Formula: cardIndex % cardBackgroundColors.size
- Updates on: Every index change
- Smooth transition: Via graphicsLayer

Data Structure:
- flashcards: List<Flashcard>
- currentIndex: Int (0-based position)
- isFlipped: Boolean (front/back)
- deck: Deck object for editing

Edit Dialog:
- Trigger: Pencil icon tap
- Input: OutlinedTextField for new name
- Validation: Non-empty required
- Persist: Via onDeckUpdate callback

7. Models.kt

Data Classes:

Deck:
  - id: String (UUID, default "")
  - name: String (unique identifier)
  - cardCount: Int (total flashcards)
  - progress: Int (percentage 0-100)
  - icon: String (emoji or unicode)
  - flashcards: List<Flashcard> (card data)

Flashcard:
  - question: String (prompt text)
  - answer: String (response text)
  - category: String (grouping/metadata)

Session:
  - deckName: String (associated deck)
  - date: String (study date)
  - duration: String (study time)
  - cardsStudied: Int (count)
  - icon: String (emoji)

DeckWithFlashcards:
  - deck: Deck
  - flashcards: List<Flashcard>

GESTURE SPECIFICATIONS
======================

Swipe Detection:

Parameters:
- Gesture type: Horizontal drag
- Minimum movement: 100dp
- Direction detection: dragAmount sign
- Continuous tracking: During drag

Animation Response:

Start Animation:
- Trigger: Swipe completion
- Calculate direction: if dragAmount > 100 (next) else (prev)
- Check bounds: if currentIndex valid
- Launch coroutine: scope.launch

Animation Execution:
- Move card: offsetX to ±1200dp
- Rotate: rotationZ to ±25 degrees
- Scale: 0.9x
- Fade: 0.7 alpha
- Duration: 400ms
- Easing: Tween (linear)

Post-Animation:
- Update index: currentIndex++/--
- Reset transforms: snapTo(0)
- Reset flip: isFlipped = false
- No gesture during animation

Tap Detection:

Parameters:
- Trigger: Single tap on card
- Action: Toggle isFlipped
- Animation: 500ms rotation Y

Button Clicks:

Parameters:
- Clickable modifier applied
- onClick callback invoked
- Ripple effect shown
- No multi-click handling needed

DEPLOYMENT INSTRUCTIONS
=======================

Installation via ADB:
1. Build: ./gradlew assembleDebug
2. Connect device: adb devices
3. Install: adb install app/build/outputs/apk/debug/app-debug.apk
4. Run: adb shell am start -n com.ramsalapps.flashcards/.MainActivity

Installation via Android Studio:
1. Open project
2. Select Run Configuration
3. Choose target device/emulator
4. Click Run button
5. Wait for build and deployment

Testing Protocol:
1. Launch app
2. Navigate to Import screen
3. Select CSV file
4. Review cards
5. Create deck
6. Study cards
7. Test swipe gestures
8. Test edit functionality
9. Close and reopen app
10. Verify data persistence
